Welcome to FayePI!

FayePI is a "fake api". It offers API endpoints you can use to build dynamic webpages and practice AJAX requests and promises with fetch.

Head over to the documentation tab to see a list of endpoints.

Fake API?

FayePI offers real endpoints that will return JSON data. What makes it "fake"? The data is just silly stuff like romantic comedies, but it's enough for practice. And the silliness makes it fun!

Documentation


Romantic Comedies

  • GET https://fayepi.herokuapp.com/romcoms

    This endpoint will return all the romantic comedies from FayePI. It does not require any parameters.

  • GET https://fayepi.herokuapp.com/romcoms/year/{year}

    This endpoint will return all romantic comedies from a given year. The year is passed in as a URL parameter. If there are no movies from the specified year in FayePI, the result will be an empty array.

  • GET https://fayepi.herokuapp.com/romcoms/rating/{rating}

    This endpoint will return all romantic comedies with the specified rating, which is passed in as a URL parameter. The rating must be in the correct format: G, PG, PG-13, R. If there is no movie with that rating, the result will be an empty array.

  • GET https://fayepi.herokuapp.com/romcoms/title/{title}

    This endpoint will return all romantic comedies with the specified title, which is passed in as a URL parameter. If there is no movie with that title, the result will be an empty array.

Nobel Prizes

  • GET https://fayepi.herokuapp.com/nobelprize

    This endpoint will return all the Nobel Prizes from FayePI. It does not require any parameters.

  • GET https://fayepi.herokuapp.com/nobelprize/year/{year}

    This endpoint will return all Nobel Prizes from a given year. The year is passed in as a URL parameter. If there are no prizes from the specified year in FayePI, the result will be an empty array.

  • GET https://fayepi.herokuapp.com/nobelprize/category/{category}

    This endpoint will return all Nobel Prizes with the specified category, which is passed in as a URL parameter. If there is no prize with that category, the result will be an empty array.

  • GET https://fayepi.herokuapp.com/nobelprize/laureateFirstName/{laureateFirstName}

    This endpoint will return all Nobel Prizes with the specified first name, which is passed in as a URL parameter. If there is no prize with that first name, the result will be an empty array.

  • GET https://fayepi.herokuapp.com/nobelprize/laureateLastName/{laureateLastName}

    This endpoint will return all Nobel Prizes with the specified last name, which is passed in as a URL parameter. If there is no prize with that last name, the result will be an empty array.

U.S. Population 1950-1959

  • GET https://fayepi.herokuapp.com/population/decade/1950

    This endpoint will return all of the U.S. population data from 1950-1959. It does not require any parameters.

  • GET https://fayepi.herokuapp.com/population/decade/1950/year/{year}

    This endpoint will return all of the U.S. population data from a given year between 1950-1959. It requires a single parameter -- the year (a number) -- which is passed in as a URL parameter.

  • GET https://fayepi.herokuapp.com/population/decade/1950/age/{age}

    This endpoint will return all of the U.S. population data between 1950-1959 for a given age. It requires a single parameter -- the age (a number) -- which is passed in as a URL parameter.

FayePI AJAX request

The code below is using an AJAX request to grab data from FayePI. Any changes to the response would take place in the if statement.


                    
                      var request = new XMLHttpRequest();
                      request.open('GET', 'https://fayepi.herokuapp.com/romcoms', true);

                      request.onload = function() {
                        if (request.status >= 200 && request.status < 400) {
                          var data = JSON.parse(request.responseText);
                        // now that we have our data, we can do something with it, like render it to a page
                        } else {
                        // we would handle for any errors here
                        }
                      };

                      request.onerror = function() {
                        // you might log here that there was an error or let the user know there's been an error.
                      };

                      request.send();

FayePI window.fetch

The code below is using a fetch request which returns a promise. You'll notice in our first function we are using .json(), which also returns a promise. Any changes we would want to make to the response would take place in the second function, where we are logging the response.

We're using the .json() function because fetch returns a response object, but what we really want is a JSON object with the data from FayePI in it. .json() does the trick!


                      
                          window.fetch('https://fayepi.herokuapp.com/romcoms')
                          .then(function(response) {
                            response.json()
                              .then(function(response2) {
                                console.log(response2);
                                })
                          });
